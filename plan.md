# Project Plan

## Goals
- Create a GitHub repository that demonstrates proper version control usage
- Practice using commits, branches, pull requests, and issues
- Build a clean and organized README and project structure

## Needs
- GitHub account and repository setup
- GitHub Desktop or Git CLI
- Basic understanding of commits, branches, and pull requests
- Time to test and update files

## Steps
1. Initialize the repository and create a README
2. Create a development branch (dev)
3. Make changes and commit with meaningful messages
4. Create and manage issues
5. Link commits to issues when fixing them
6. Open pull requests and merge changes into main
7. Update README with full project details
8. Review repository to ensure everything meets requirements
