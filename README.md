# HW 4.4 – GitHub Practice

## Overview
This repository was created for HW 4.4 to practice using GitHub features such as branching, commits, pull requests, and issue tracking.

## Purpose
The goal of this project is to demonstrate understanding of version control workflows, including creating branches, making meaningful commits, linking commits to issues, and merging changes through pull requests.

## Data
This project does not use any external data.

## Repository Structure
- `README.md` – Project description and documentation  
- Additional changes were made through branches and commits to simulate a real development workflow

## Current Progress
All required GitHub features have been completed, including:
- Creating and managing branches
- Making commits with meaningful messages
- Creating and closing issues
- Merging a pull request into the main branch.
